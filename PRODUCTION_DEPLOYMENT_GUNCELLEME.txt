## 🚀 HIZLI DEPLOYMENT (ÖNERİLEN)

### 1. Sunucuya Bağlanma:
```bash

# veya SSH etkinleştirildikten sonra:

```

### 2. Otomatik Deployment:
```bash
# Proje dosyalarını kopyala
scp -r bulutistan/ ubuntu@10.40.39.24:/var/www/

# Sunucuda deployment'ı başlat
cd /var/www/bulutistan
chmod +x deploy.sh
./deploy.sh
```

### 3. Manuel Kontrol:
```bash

sudo systemctl status bulutistan
sudo systemctl status nginx

# Admin paneli testi

```

## 🔍 ADMIN PANELİ SORUN GİDERME

### Admin Paneli Açılmıyorsa:

#### 1. Django Servisi Kontrolü:
```bash
# Gunicorn durumu
sudo systemctl status bulutistan

# Logları kontrol et
sudo journalctl -u bulutistan -f

# Socket dosyası kontrolü
ls -la /var/www/bulutistan/bulutistan.sock
```

#### 2. Nginx Kontrolü:
```bash
# Nginx durumu
sudo systemctl status nginx

# Konfigürasyon testi
sudo nginx -t

# Logları kontrol et
sudo tail -f /var/log/nginx/error.log
```

#### 3. Veritabanı Kontrolü:
```bash
# PostgreSQL durumu
sudo systemctl status postgresql

# Veritabanı bağlantısı
sudo -u postgres psql -c "\l"
```

#### 4. Django Test:
```bash
cd /var/www/bulutistan
source venv/bin/activate
export DJANGO_SETTINGS_MODULE=bulutistan.settings_production

# Django check
python manage.py check --deploy

# Migration kontrolü
python manage.py showmigrations
```

## 📞 DESTEK VE YARDIM

### Yaygın Sorunlar:

#### 1. "502 Bad Gateway" Hatası:
- Gunicorn servisi çalışmıyor
- Socket dosyası oluşmamış
- Nginx konfigürasyonu hatalı

#### 2. "500 Internal Server Error" Hatası:
- Django ayarları hatalı
- Veritabanı bağlantı sorunu
- Static dosya yolu hatalı

#### 3. "404 Not Found" Hatası:
- URL yapılandırması hatalı
- Template dosyası bulunamıyor
- Static dosya yolu yanlış

### Çözüm Adımları:
1. Log dosyalarını kontrol et
2. Servis durumlarını kontrol et
3. Konfigürasyon dosyalarını kontrol et
4. Gerekirse servisleri yeniden başlat
